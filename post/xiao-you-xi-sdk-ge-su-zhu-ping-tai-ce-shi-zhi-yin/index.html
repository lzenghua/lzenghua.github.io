小游戏聚合 SDK 各平台测试指引

1. 环境

  • node 版本>=8.9.0
  • adb工具 地址
  • 手机开启开发者模式，打开 usb 调试
  • 手机与电脑处于同一局域网

2. oppo

  • egret 引擎发布 -> oppo 小游戏（修改设置->最小平台版本号 1040）

  • 打包工具：

    • 下载解压quickgame-toolkit
      # 进入解压目录
  cd quickgame-toolkit
  # 安装项目的依赖
  npm install
    • 将 quickgame-toolkit 下的"/lib/bin"目录加入系统变量中（例如：/path/quickgame-toolkit/lib/bin，其中"/path/quickgame-toolkit"是解压安装的目录）
    • 运行 quickgame -V 能够正确显示版本号表示安装成功:如果不成功的话，使用 powershell 执行
      # 打开 powershell
  # cd 到发布后小游戏的目录下
  cd C:\Users\Administrator\Desktop\miniGameTest\test_vivogame
  # 执行打包命令
  quickgame pack

  • 安装runtime.apk包到 oppo 手机上小游戏调试器；

  • 在手机的 sd 卡，将小游戏 apk 包放在此电脑\OPPO R9sk\内部存储设备\Android\data\com.nearme.instant.platform\files\games中；如没有games目录则手动新增

  • chrome 真机调试：

# 注：新版本 chrome 83 需要去掉 chrome-前缀
# 其中 {IP} 为手机本地 IP，{PORT} 为端口号，端口号有 12345 和 12346，如果 12345 连接不上，请使用 12346。
chrome-devtools://devtools/bundled/inspector.html?v8only=true&ws={ip}:{PORT}/00010002-0003-4004-8005-000600070008

3. vivo

  • 在 vivo 真机上安装小游戏调试器;小游戏引擎
  • 安装调试环境：npm install -g @vivo-minigame/cli; mg -v 查看是否安装成功
  • egret 中发布为 vivo 小游戏 -> cd test_vivogame目录 -> 执行npm run server会生成一个二维码，然后使用小游戏调试器扫码安装游戏 apk 包；
  • 打包 apk 包: npm run build
  • 也可以在本地安装 apk 包，将 apk 包传到手机 sd 卡执行安装
  • chrome 真机调试：
devtools://devtools/bundled/inspector.html?v8only=true&ws={手机IP}:5086/00010002-0003-4004-8005-000600070008

4. 微信小游戏

  • 安装微信开发者工具
  • 修改egretProperties.json中的：
"target": {
  "current": "wxgame"
},

然后执行调试；或者发布为微信小游戏 -> 使用微信小游戏工具打开

  • 调试的时候，如果使用的是测试 appId,那么会遇到请求不合法的情况，可以在工具右上角的：详情 -> 本地设置 -> 勾选“不检验合法域名”
